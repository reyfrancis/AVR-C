In C++ we have a statement called if statement. In this statement, if the condition mentioned is true then a particular set of statements are executed else those statements are neglected and the compiler doesn't execute them.

The syntax is:

if( test expression )

{

Statements;

}

In the language of computers 0 means false and any other number means true. So if you write 0 as a test expression then the compiler doesn't execute the statements written in the braces. But if you mention any other number, then the statements are executed.

In C ,C++ and in most programming languages , 0 is considered false , or directly binary 0 or logical 0 , and any high value , actually even lower values that are not equal to 0 are considered true .

Hence if(3) means if 3 is true , or if 3 is not 0

And f(0) means if 0 is not 0.

Well since 3 is not 0 always true so if condition always execute

On the other hand since 0 is always 0 , so 0 is not 0 is a false statement hence it never executes

if(3) means if(3>0)

In C++, if we take any numeric value in condition then compiler automatically compares it for greater than 0

Eg:

if(3) -> if(True)

if(-1) -> if(False)

But it is not recommended, so you should avoid it while writting code as it will compile but theoretically for a person reading the code it doesn't have any direct meaning(Your code should always be readable and easily understandable)
