As far as I know, yes you will need to do them separately.

You can make your life easier by defining

Code: [Select]
#define COMMAND(c, p, q) { Serial.write(c); Serial.write(p); Serial.write(q); }

If there are timing issues (ie inter character pauses while multiple calls execute), then an alternative

Code: [Select]
void command(char c, char p, char q)
{
  char cmd[4];

  cmd[0]=c;
  cmd[1]=p;
  cmd[2]=q;
  cmd[3]='\0';

  Serial.print(cmd);
}

The Serial.write() method is overloaded. One version takes a single value. The other takes an array and a length.

You could use:
Code: [Select]
Serial.write(conveyerCommand);
Serial.write(conveyer1);
Serial.write(relayOn);

or:
Code: [Select]
byte stuff[] = { conveyerCommand, conveyer1, relayOn };
Serial.write(stuff, sizeof(stuff));

The macro method needs to be rewritten to change the number of values. The array method does not (except for adding more data to the array).
