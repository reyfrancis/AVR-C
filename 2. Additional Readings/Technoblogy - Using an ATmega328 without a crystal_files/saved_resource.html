          Mike, K8LH • 10 months ago • edited

          David, I've been using your "ATmegaBreadboard" package (thank you) and the "ATmega328 on a breadboard (8 MHz internal clock)" package, when I need a bootloader installed. However, I just found MCUdude's MiniCore - An Arduino core for the ATmega8/ 48/ 88/ 168/ 328, which seems to include most of the features of both packages.

          The MiniCore package supports the same '168 and '328 variants in your ATmegaBreadboard package as well as the newer 'PB' variants of those chips. It also provides a "yes/no" option for installing a 512 byte OptiBoot bootloader and it defines Arduino digital pins 20 (PB6) and 21 (PB7) when using the internal oscillator (at the cost of 72 bytes of program memory). Not as many oscillator options as in ATmegaBreadboard but still quite usable.

                johnsondavies Mod Guest • a year ago

                Once the chip has been programmed to use a crystal, you need to have a crystal there to change the fuses to make it work without a crystal. Do you have a crystal and a couple of capacitors you could try it with? It doesn't have to be 16 MHz.

                Alternatively, do you have a brand new ATmega328P you can try, which has never been programmed on the Arduino Uno?

                Alternatively you should be able to program a chip while it's in the Uno board, using the Uno's crystal, by connecting the wires from the USBtinyISP to the appropriate pins on the Uno.
